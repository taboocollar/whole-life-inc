## Usage Guidelines

### For Developers

1. **Load Persona Configuration**: Import the appropriate `persona.json` file
2. **Review Documentation**: Read the persona's `.md` file thoroughly
3. **Implement Safety Systems**: Ensure all consent and boundary protocols are active
4. **Initialize Properly**: Follow the initialization protocol in the documentation
5. **Monitor Interactions**: Regular safety audits and quality assessments

### For Users

1. **Understand the Persona**: Read the character documentation
2. **Set Boundaries**: Clearly communicate your comfort levels
3. **Use Safewords**: Familiarize yourself with halt mechanisms ("pause", "reset", "stop")
4. **Provide Feedback**: Help improve the persona through honest feedback
5. **Respect Safety**: These personas prioritize consent and wellbeing

## Safety & Ethics

All personas in this directory adhere to strict safety guidelines:

✓ **Consent-First**: Multi-stage consent verification for intimate content  
✓ **Clear Boundaries**: Explicit prohibition of harmful, non-consensual, or illegal content  
✓ **User Control**: Safewords, check-ins, and ability to redirect at any time  
✓ **Transparency**: Open about AI nature and limitations  
✓ **Privacy**: Clear data handling and retention policies

## Adding New Personas

To add a new persona to this directory:

1. Create a subdirectory with the persona name (kebab-case)
2. Include required files:
   - `persona.json` - Complete configuration
   - `persona.md` - Full documentation
   - `README.md` - Quick start guide
3. Follow established safety protocols
4. Update this README with persona details
5. Version all files appropriately
